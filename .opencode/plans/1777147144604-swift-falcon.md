# Plan: Add 4K@120fps Available Mode to Headless Sway Session

## Goal
Add 4K@120fps as an available display mode for the headless Sway session (`HEADLESS-1`) used by Sunshine game streaming, while keeping 1080p@60Hz as the default. This enables streaming at higher resolutions when the client supports it.

## Why
- Current config uses `resolution 1920x1080@60Hz` which is technically incorrect â€” `resolution` only accepts `WxH`, not refresh rate. This is a bug fix.
- The RTX 5090 supports 4K@120 NVENC encoding. Adding the mode enables higher-quality streams.
- 1080p remains default to conserve bandwidth for typical streaming scenarios.

## Changes

### File: `sway-sunshine/config` (lines 7-8)

**Before:**
```
output HEADLESS-1 enable
output HEADLESS-1 resolution 1920x1080@60Hz
```

**After:**
```
output HEADLESS-1 enable
output HEADLESS-1 mode 1920x1080@60Hz
output HEADLESS-1 mode 3840x2160@120Hz preferred
```

**Rationale:**
- `resolution` â†’ `mode`: The `resolution` directive in Sway only accepts `WxH`. Using `mode` properly includes the refresh rate.
- `preferred` on 4K@120: Without it, `mode` directives in Sway select the last one listed as active. However, `set-resolution.sh` overrides via `swaymsg` at runtime, so the initial mode doesn't matter for streaming. The `preferred` keyword is kept for clarity.
- Both modes are registered in Sway's mode list. `swaymsg "output HEADLESS-1 mode <mode>"` can switch between them at runtime.

### No changes needed to:
- `sway-sunshine/set-resolution.sh` â€” already uses dynamic `swaymsg "output HEADLESS-1 mode ${SUNSHINE_CLIENT_WIDTH}x${SUNSHINE_CLIENT_HEIGHT}@${SUNSHINE_CLIENT_FPS}Hz"`, which works for any registered mode including 4K@120.
- `sway-sunshine/reset-resolution.sh` â€” already resets to 1920x1080@60Hz.

### No changes needed to:
- `sway-sunshine/service` files â€” no env var or command changes required.
- `sunshine/sunshine.conf` â€” encoder selection is automatic; NVENC on RTX 5090 handles 4K@120.
- `apps.json` â€” prep-cmd hooks already use the dynamic resolution scripts.

## Deployment

1. **Install (template & copy):**
   ```bash
   ./install.sh
   ```
   This templates the user ID into scripts and copies `sway-sunshine/config` to `~/.config/sway-sunshine/config`.

2. **Restart services:**
   ```bash
   systemctl --user restart sway-sunshine.service sunshine-headless.service
   ```

## Verification Steps

### 1. Check Sway config loaded without errors
```bash
journalctl --user -u sway-sunshine.service -n 30 --no-pager | grep -iE 'error|fail'
```
Expected: No relevant errors.

### 2. Verify both modes are registered
```bash
SWAYSOCK=/run/user/1000/sway-sunshine.sock swaymsg -t get_outputs | jq '.[] | select(.name == "HEADLESS-1") | {current_mode, modes}'
```
Expected: `current_mode` shows `1920x1080@60Hz`, `modes` array contains both `1920x1080@60Hz` and `3840x2160@120Hz`.

### 3. Test manual switch to 4K@120
```bash
SWAYSOCK=/run/user/1000/sway-sunshine.sock swaymsg "output HEADLESS-1 mode 3840x2160@120Hz"
SWAYSOCK=/run/user/1000/sway-sunshine.sock swaymsg -t get_outputs | jq '.[] | select(.name == "HEADLESS-1") | .current_mode'
```
Expected: `"3840x2160@120Hz"`

### 4. Test reset back to 1080p
```bash
SWAYSOCK=/run/user/1000/sway-sunshine.sock swaymsg "output HEADLESS-1 mode 1920x1080@60Hz"
```
Expected: `current_mode` returns to `1920x1080@60Hz`.

### 5. Test stream via Moonlight CLI
```bash
flatpak run com.moonlight_stream.Moonlight stream localhost "Desktop"
```
- In Moonlight, select 4K@120 as streaming resolution if prompted.
- Verify stream connects with video (not black screen).
- Check NVENC is active: `journalctl --user -u sunshine-headless.service | grep -i nvenc`

### 6. Disconnect stream and verify auto-reset
```bash
SWAYSOCK=/run/user/1000/sway-sunshine.sock swaymsg -t get_outputs | jq '.[] | select(.name == "HEADLESS-1") | .current_mode'
```
Expected: `1920x1080@60Hz` (reset by `prep-cmd.undo`).

## Potential Issues & Mitigations

| Issue | Likelihood | Mitigation |
|-------|-----------|------------|
| wlroots only registers the last `mode` directive | Low | Fallback: remove both `mode` lines, rely entirely on runtime `swaymsg` switching via `set-resolution.sh` |
| 4K@120 requires ~100-200 Mbps bitrate | Medium | Verify network connection (Ethernet recommended). Adjust bitrate in Sunshine if needed. |
| NVENC struggles with 4K@120 on current driver | Low | RTX 5090 Ada Lovelance supports it natively. If issues, try `WLR_RENDERER=vulkan` in `sway-sunshine.service`. |
| Moonlight client doesn't offer 4K@120 option | Low | Depends on Moonlight version and client capabilities. Desktop client supports it. |
