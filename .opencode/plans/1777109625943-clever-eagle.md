# Headless Isolation + KMS Capture on KDE Plasma Wayland

## Goal
Fix the broken headless Sway + Sunshine setup on KDE Plasma Wayland, preserving full isolation between gaming (headless session) and normal desktop use.

## Root Causes Found

1. **sway-sunshine crashes on startup** — `WLR_DRM_DEVICES=/dev/dri/renderD128` pointed to NVIDIA GPU (card0). Fixed by changing to `renderD129` (AMD).
2. **capture = wlr doesn't work on KDE Plasma** — KWin doesn't implement wlroots screencopy protocol. Fixed by switching to `capture = kms`.
3. **Cross-GPU DMA-BUF crash** — KMS captures from AMD GPU, but Vulkan encoder runs on NVIDIA GPU (RTX 5090). Cross-device DMA-BUF import fails → SEGV crash → "No video received."

## Key Hardware Mapping

| Device Node | GPU                    | Vendor  | PCI Address   |
| ----------- | ---------------------- | ------- | ------------- |
| card0 / renderD128 | NVIDIA RTX 5090 | 0x10de  | 0000:c2:00.0  |
| card1 / renderD129 | AMD Radeon 8060S | 0x1002 | 0000:03:00.0  |

Only **DP-4** is connected on the AMD GPU (main display). HDMI-A-1 also shows as "connected."

## Files Modified

| File | Change |
|------|--------|
| `~/.config/systemd/user/sway-sunshine.service.d/override.conf` | `WLR_DRM_DEVICES=/dev/dri/renderD129` (AMD) |
| `~/.config/systemd/user/sunshine-headless.service.d/override.conf` | KWIN env vars + **VK_ICD_FILENAMES for AMD only** |
| `~/.config/sunshine/sunshine.conf` | `capture = kms`, `adapter_name = /dev/dri/card1` |

### Phase 2: Fix cross-GPU DMA-BUF crash (IN PROGRESS)
- [ ] **2.1** Root cause: Sunshine has two Vulkan ICDs (NVIDIA + AMD/RADV). It picks NVIDIA first, but NVIDIA's Vulkan can't import DMA-BUF from AMD's KMS capture → SEGV crash.
- [ ] **2.2** Fix: Add `VK_ICD_FILENAMES=/usr/share/vulkan/icd.d/radeon_icd.json` to sunshine-headless override so Sunshine only sees AMD's Vulkan driver (RADV).
  ```bash
  cat > ~/.config/systemd/user/sunshine-headless.service.d/override.conf << 'EOF'
   [Service]
   Environment=KWIN_DRM_NO_DIRECT_SCANOUT=1
   Environment=KWIN_FORCE_SW_CURSOR=1
   Environment=VK_ICD_FILENAMES=/usr/share/vulkan/icd.d/radeon_icd.json
   EOF
  ```
- [ ] **2.3** Reload and restart:
  ```bash
  systemctl --user daemon-reload
  systemctl --user restart sunshine-headless.service
  sleep 5
  journalctl --user -u sunshine-headless -n 30 --no-pager
  ```
- [ ] **2.4** Verify logs show Vulkan encoding on AMD GPU (not NVIDIA):
  Look for: `Vulkan encode using GPU: AMD Radeon ...` instead of `NVIDIA GeForce RTX 5090`
- [ ] **2.5** Test with Moonlight — should no longer crash
