## NEW ISSUE: Audio Sink Race Condition Fix

### Root Cause
`restore-default-sink.sh` has a race condition:
1. Captures original default sink ID at prep-cmd start
2. Sunshine sets `audio_sink = sink-sunshine-stereo` as system default
3. Restore script detects this change and immediately restores original sink (Ryzen HD)
4. This interrupts Sunshine's audio capture initialization

### Fix: Modify `/home/moe/sunshine-headless-sway/sway-sunshine/restore-default-sink.sh`

**Current logic (line 19-20):**
```bash
systemd-run --user --no-block --unit=sunshine-sink-restore \
    bash -c 'for i in $(seq 1 30); do sleep 1; CUR=$(wpctl status 2>/dev/null | grep -A20 "Sinks:" | grep "\*" | head -1 | grep -oE "[0-9]+" | head -1); if [ "$CUR" != "'"$SINK_ID"'" ] && [ -n "$CUR" ]; then wpctl set-default '"$SINK_ID"'; exit 0; fi; done'
```

**New logic:** Check if the new default is `sink-sunshine-stereo` (Sunshine's null sink) and skip restoration in that case. Only restore if the default changes to something unexpected.

Replace the entire file with:
```bash
#!/bin/bash
# Restores the host's default audio sink after Sunshine changes it.
# Sunshine sets audio_sink as the system default when a client connects.
# Uses systemd-run to spawn a detached watcher that survives prep-cmd cleanup.
# Skips restoration if the new default is sink-sunshine-stereo (Sunshine's null sink).

# Get the current default sink ID from wpctl
# Format: " │  *   44. Komplete Audio 2 ..."
SINK_ID=$(wpctl status 2>/dev/null | grep -A20 'Sinks:' | grep '\*' | head -1 | grep -oE '[0-9]+' | head -1)

if [ -z "$SINK_ID" ]; then
    exit 0
fi

# Clean up any previous instance
systemctl --user stop sunshine-sink-restore 2>/dev/null
systemctl --user reset-failed sunshine-sink-restore 2>/dev/null

# Launch a detached watcher via systemd-run (survives prep-cmd cleanup)
systemd-run --user --no-block --unit=sunshine-sink-restore \
    bash -c 'for i in $(seq 1 30); do sleep 1; CUR=$(wpctl status 2>/dev/null | grep -A20 "Sinks:" | grep "\*" | head -1 | grep -oE "[0-9]+" | head -1); if [ "$CUR" != "'"$SINK_ID"'" ] && [ -n "$CUR" ]; then CUR_NAME=$(wpctl get-info sink.$CUR 2>/dev/null | grep "Name:" | cut -d: -f2 | xargs); if [ "$CUR_NAME" = "sink-sunshine-stereo" ]; then continue; fi; wpctl set-default '"$SINK_ID"'; exit 0; fi; done'
```

**Key change:** Added a check `CUR_NAME=$(wpctl get-info sink.$CUR 2>/dev/null | grep "Name:" | cut -d: -f2 | xargs); if [ "$CUR_NAME" = "sink-sunshine-stereo" ]; then continue; fi` before restoring. If the new default is Sunshine's null sink, skip restoration and continue polling.
