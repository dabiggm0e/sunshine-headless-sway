# Implementation Plan: Add Lutris Launcher Support to LutrisToSunshine

## Goal
Add Lutris games to the Sunshine games list by parsing game config files directly (no Lutris CLI dependency) and building headless-appropriate launch commands.

## Context & Decisions
| Decision | Rationale | Source |
|----------|-----------|--------|
| Parse YAML configs directly instead of `lutris -lo --json` | `lutris -lo --json` fails in `uv run` isolated venv because Lutris can't find its Python modules; parsing YAML is environment-independent | `ref:ses_23a0dd22bffe3gPR2CDhx70DbA`, `ref:ses_23a07e9ddffesP2mxmRuhpLZlC` |
| Use `wine` binary directly from Wine prefixes for headless launch | `lutris:rungameid/{id}` URI scheme requires Lutris GUI running; direct Wine invocation is headless-appropriate | `ref:ses_23a07e9ddffesP2mxmRuhpLZlC` |
| Return enriched metadata dict from `list_lutris_games()` | Enables `build_game_command()` to construct runner-specific commands from parsed config fields (exe, prefix, args, wine version) | `ref:ses_23a0dd22bffe3gPR2CDhx70DbA` |
| Support wine, linux, and steam runners; skip emulators | Wine is most common; linux runners are simple executables; steam runners use `steam://run/`; emulators (rpcs3, ppsspp, libretro) require Lutris script engine which is too complex for direct invocation | `ref:ses_23a07e9ddffesP2mxmRuhpLZlC` |
