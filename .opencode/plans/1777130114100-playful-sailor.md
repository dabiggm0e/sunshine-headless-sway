# Plan: Stale Sway Session Detection + WAYLAND_DISPLAY Templating

## Problem

1. **Stale zombie sockets**: Crashed sway-sunshine sessions leave behind `wayland-*` sockets that confuse install.sh's auto-detection heuristic (`sort | tail -1`).
2. **Service files not templated**: Both `sway-sunshine.service` and `sunshine-headless.service` hardcode `WAYLAND_DISPLAY=wayland-1`. install.sh does NOT template this variable into service files (only templates UID paths and ExecStart).
3. **No PID tracking**: No way to detect/stop a running sway-sunshine session from outside systemd.

## Changes

### Change 1: New file — `sway-sunshine/sway-wrapper.sh`

A thin wrapper that writes its PID to a file before exec'ing sway:

```bash
#!/bin/bash
RUNTIME_DIR="/run/user/$(id -u)"
PID_FILE="$RUNTIME_DIR/sway-sunshine.pid"

echo $$ > "$PID_FILE"
trap 'rm -f "$PID_FILE"' EXIT

exec /usr/bin/sway --config "$HOME/.config/sway-sunshine/config"
```

- `trap ... EXIT` ensures PID file cleanup on crash/kill
- `exec` replaces wrapper process — sway inherits the PID, so PID file stays valid
- Uses `$HOME` (set by systemd) instead of hardcoded path

### Change 2: Update `systemd/sway-sunshine.service`

- Remove line: `ExecStartPre=-/bin/rm -f /run/user/1000/sway-sunshine.sock`
- Change ExecStart from `/usr/bin/sway --config %h/.config/sway-sunshine/config` to `%h/.config/sway-sunshine/sway-wrapper.sh`

### Change 3: Update `systemd/sunshine-headless.service`

No structural changes — just needs WAYLAND_DISPLAY templated by install.sh sed pass.

### Change 4: Update `install.sh` — Stale session check

Insert after line 123 (after HEADLESS_DISPLAY detection echo), before line 125 (USER_ID detection):

```bash
# Check for stale sway-sunshine session and stop it gracefully
PID_FILE="/run/user/$USER_ID/sway-sunshine.pid"
if [ -f "$PID_FILE" ]; then
    OLD_PID=$(cat "$PID_FILE")
    WAYLAND_SOCKET=""

    if kill -0 "$OLD_PID" 2>/dev/null; then
        echo "Found running sway-sunshine session (PID $OLD_PID), stopping..."

        # Scan Wayland socket BEFORE killing — /proc/PID/fd disappears after
        WAYLAND_DIR="/run/user/$USER_ID"
        WAYLAND_SOCKET=$(ls -l /proc/$OLD_PID/fd 2>/dev/null | grep "$WAYLAND_DIR/wayland-" | head -1 | sed 's|.*-> \(.*\)|\1|')

        kill -INT "$OLD_PID"
        for i in $(seq 1 10); do
            kill -0 "$OLD_PID" 2>/dev/null || break
            sleep 1
        done
        if kill -0 "$OLD_PID" 2>/dev/null; then
            echo "Session did not stop gracefully, force killing..."
            kill -9 "$OLD_PID" 2>/dev/null
            sleep 1
        fi
    else
        echo "Found stale PID file (process $OLD_PID not running)"
    fi

    rm -f "$PID_FILE" "$SOCKET_PATH"

    # Remove Wayland socket — use /proc scan result, fall back to filesystem
    WAYLAND_DIR="/run/user/$USER_ID"
    if [ -n "$WAYLAND_SOCKET" ] && [ -e "$WAYLAND_SOCKET" ]; then
        rm -f "$WAYLAND_SOCKET"
        echo "Removed sway-sunshine wayland socket: $WAYLAND_SOCKET"
    else
        # Fallback: remove highest-numbered wayland socket if multiple exist
        WAYLAND_COUNT=$(ls "$WAYLAND_DIR"/wayland-* 2>/dev/null | grep -v lock | wc -l)
        if [ "$WAYLAND_COUNT" -ge 2 ]; then
            HIGH=$(ls "$WAYLAND_DIR"/wayland-* 2>/dev/null | grep -v lock | sort | tail -1)
            rm -f "$HIGH"
            echo "Removed stale wayland socket: $(basename "$HIGH")"
        fi
    fi

    echo "Previous session stopped."
fi
```

**Key fix:** The `/proc/$PID/fd` scan happens **before** `kill -INT`, because the fd entries disappear once the process is killed. The old code scanned after killing, so it always found nothing and never cleaned up the Wayland socket.

### Additional Fix: pipefail-safe grep pipelines

The script uses `set -euo pipefail`. With `pipefail`, if `grep` finds no matches it returns exit code 1, causing the entire pipeline to fail and `set -e` to kill the script. Two pipelines need `|| true`:

**Line 55 (in Change 4 block):**
```bash
        WAYLAND_SOCKET=$(ls -l /proc/$OLD_PID/fd 2>/dev/null | grep "$WAYLAND_DIR/wayland-" | head -1 | sed 's|.*-> \(.*\)|\1|' || true)
```
⚠️ **CRITICAL:** The sed replacement must be `\1|` (pipe delimiter), NOT `\1//'` (double-slash). A previous agent incorrectly changed it to `\1//'` which would corrupt the path.

**Line 80 (in Change 4 block):**
```bash
        WAYLAND_COUNT=$(ls "$WAYLAND_DIR"/wayland-* 2>/dev/null | grep -v lock | wc -l || true)
```
